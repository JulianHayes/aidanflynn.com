# Pre-Sequence Prompts

> Part 1 of 6 in the Brand Schema Master Prompt Library

Run these before the main sequence. They front-load critical thinking and set decision frameworks.

---

## Red Team

Pre-sequence, Run 1 of 3. Foundation. Run this before Prompt 1.

| Setting | Value |
|---|---|
| Conversation | New conversation (solo prompt) |
| Web Search | OFF |
| Deep Research | OFF |
| Output | 1500-2000 word critique document |
| Time | 30 minutes |

### Context

```text
Brand Schema is a two-person AI-augmented verbal identity consultancy.
Thesis: brands lacking distinct verbal identity are invisible to AI engines.
Founders: Julian (methodology), Justin (commercial).
Target sectors: financial services, B2B technology, automotive, retail, luxury, FMCG.
They have completed foundational research and are about to run an 11-prompt strategic sequence.
```

### Prompt

```text
Before they begin the main strategic build, act as a critical advisor.

Tell me:
1. What are the 10 things most founders forget to address when building this type of consultancy?
2. What would the top 1% of consultancy founders do that this team is unlikely to think of?
3. What questions should they be able to answer before they start that they probably cannot answer yet?
4. Looking at their stated target sectors and positioning, what are the most dangerous blind spots?
5. What financial, operational, or market timing risks are they missing?

Do not be polite. Be useful. Flag anything that would change their direction if they knew about it now.
```

## Checkpoint

Pre-sequence, Run 2 of 3. Mid-point review. Run this after Prompt 6 (Buyer Simulation).

| Setting | Value |
|---|---|
| Conversation | Same conversation as Prompt 6 |
| Web Search | OFF |
| Deep Research | OFF |
| Output | 800-1200 word gap analysis |
| Time | 60 minutes |

### Context

```text
You have completed six foundational prompts:
1. Competitive Landscape (Prompt 1)
2. Demand Analysis (Prompts 2a + 2b)
3. Market Sizing (Prompts 3a + 3b)
4. Positioning Panel (Prompts 4a, 4b, 4c)
5. Service Architecture (Prompt 5)
6. Buyer Simulation (Prompt 6)

Before continuing with messaging, verbal identity, stress testing, and launch planning, conduct a gap review.
```

### Prompt

```text
Tell me what is missing from this body of work that will be obvious in hindsight.

Focus on:
1. Unvalidated assumptions being treated as facts
2. Financial gaps or cash flow realities not modelled
3. Whether the buyer identification is correct (is it really the CMO, or Head of Content/SEO?)
4. Go-to-market timing and velocity risks
5. Anything that would change the service architecture if true

Do not ask for clarification. Make your best assessment based on what has been produced.
```

## Kill Criteria

Pre-sequence, Run 3 of 3. Decision gates. Run this after Prompt 9b (Pre-Mortem).

| Setting | Value |
|---|---|
| Conversation | Same conversation as Prompt 9b |
| Web Search | OFF |
| Deep Research | OFF |
| Output | 1200-1600 word decision framework |
| Time | 60 minutes |

### Context

```text
You have completed the pre-mortem and stress test (Prompt 9b). You identified five failure modes and their probability estimates.

Now build a decision framework: what specific conditions would cause the founders to stop and pivot vs. continue?
```

### Prompt

```text
Build a kill criteria framework for Brand Schema.

For each criterion, specify:
1. The measurable condition (specific, not vague)
2. The decision date (by when must this be true?)
3. What happens if it fails (pivot to Audit-only? Reduce team? Shut down?)
4. How you will know it has failed (specific evidence)

Generate five criteria specific to Brand Schema. Examples (do not use):
- "If we have fewer than three sales-ready pipeline conversations by 15 May 2026, reduce scope to Brand Schema Audit only"
- "If the first client engagement does not generate measurable AI visibility lift, do not pursue Tier 2 codification"

After the five criteria, answer: which criterion would actually cause the founders to pivot, and which would they push on even if it failed?
```
